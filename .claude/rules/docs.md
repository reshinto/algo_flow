## Documentation Rules

### README.md Must Document

- Project setup (npm install, npm run dev)
- Docker and Docker Compose usage
- Test commands (unit, storybook, e2e)
- Storybook usage
- CI/CD pipeline overview
- Algorithm categories and examples
- Educational content approach
- Temporary input editing behavior
- Temporary pathfinding editing behavior
- Pre-session branch hook
- Post-session quality gate
- `.claude/PLAN.md` location
- How to add a new algorithm
- How to add a new language implementation

### Code Comments

- Comment intent and purpose, not mechanics
- No bloated commentary
- JSDoc for exported functions and types
- No comments that restate what the code does

### No AI References

- Never mention Claude, AI, assistant, automation, or generated in:
  - Commits, PRs, branch names
  - README, docs, code comments
  - Any repo artifact
